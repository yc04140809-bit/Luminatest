# cost-check — API利用料が増える変更の検出

このプロジェクトの方針: 有料API・外部サービス・新しいnpmパッケージを追加する場合、実装前に「何のために必要か」「無料で代替できないか」「料金が発生する可能性」「既存環境への影響」を説明する（詳細: `.claude/rules/safety.md`）。

## 1. 変更前の確認

- 追加しようとしている機能が、Anthropic API呼び出し（`llmClient.ts`の`callTool`/`callToolWithWebSearch`）を新たに増やすか確認する
- 増やす場合、「1操作1回呼び出し」の既存原則から外れていないか（無駄なループ呼び出し・不要なリトライ多用がないか）
- Web検索ツール（`callToolWithWebSearch`）等、通常のテキスト生成より単価が高い可能性のある呼び出しを使う場合は、その旨を明示する
- 新しい有料外部サービス・npmパッケージを追加する場合、無料の代替手段がないか先に検討する（例: Chaos Video StudioはFFmpegサーバー処理ではなくブラウザ内完結方式を選び追加コストゼロにした前例がある。詳細: `.claude/rules/decisions.md`）

## 2. コードでの確認

- `git diff`で`llmClient.ts`呼び出し箇所・新規`package.json`依存を`Grep`で洗い出す
- ループ内でAI呼び出しをしていないか、同じ内容を複数回呼び出していないかを確認する
- 新しい環境変数（APIキー等）が追加されている場合、既存の`secretsStore`の仕組みに沿っているか確認する

## 3. 報告

料金が発生する可能性がある変更を見つけた場合は、実装を進める前にケイオス師匠へ「何のためか・無料代替の有無・料金発生の可能性・既存環境への影響」を日本語で説明し、確認を取ってから進める。変更後は、AI呼び出し回数が増えたかどうかを完了報告に含める。
