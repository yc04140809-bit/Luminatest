# mobile-check — スマホUI確認

想定ユーザーの多くはPCを持たずスマートフォンだけで操作する。詳細ルールは`.claude/rules/mobile-first.md`。

## 1. コードでの確認

- 横スクロールが発生しうるレイアウトがないか（固定幅・`overflow-x`）
- 文字・ボタン・入力欄・モーダル・メニューが画面幅（375px程度を基準）で切れないか
- タップ領域が目安44px四方を下回っていないか
- 縦1列（1カラム）を基本にしているか。横並びグリッドがある場合、狭い画面で崩れないか
- 新しい画面が既存パターン（サイドバーのカード → フルスクリーンモーダル`fixed inset-0 z-[70]`、`office-*`トークン、共通クラス定数`inputCls`/`labelCls`/`btnPrimary`/`btnSub`）に沿っているか

## 2. 可能であれば`mobile-ui-reviewer`サブエージェントでレビューする

`Agent`ツールで`mobile-ui-reviewer`を呼び出し、変更差分をスマホUI観点で確認させる。

## 3. 実機・ブラウザでの確認

- 開発サーバー（`npm run dev:frontend`）を起動し、ブラウザの開発者ツールでスマホ幅（375px前後）に切り替えて操作確認する
- 自動確認できるのは要素の存在・レイアウト崩れの可能性までで、実機（Android Chrome等）での最終的な見た目確認はケイオス師匠に依頼する必要があることを明示する

## 4. 報告

確認できた項目と、実機確認が必要なため「未確認」として残る項目を分けて、初心者向けの日本語で報告する。
