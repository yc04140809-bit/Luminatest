# 自動品質保証レイヤー（Hooks / Subagents / Skills）

このファイルは、コード変更のたびに人間が毎回指示しなくても最低限の品質確認が走る仕組み（自動品質保証レイヤー）の設計と使い方を記録する。2026年7月導入。

## 何を自動化しているか

| # | 確認項目 | 仕組み |
|---|---|---|
| 1 | TypeScript型エラー | Hook（編集のたび） |
| 2 | ビルドエラー | 型チェックで実質カバー（`tsc -b`は型エラー=ビルド阻害要因を検出） |
| 3 | 既存機能への影響 | `code-reviewer`サブエージェント + Stopフックのバックエンドテスト |
| 4 | セキュリティ上の危険な変更 | `security-reviewer`サブエージェント |
| 5 | スマホ表示の崩れ | `mobile-ui-reviewer`サブエージェント |
| 6 | 修正内容の記録 | 通常のcommitメッセージ + `.claude/rules/decisions.md`（重要な設計判断のみ） |

## Hooks（`.claude/settings.json` + `.claude/hooks/`）

- **PostToolUse（`post-edit-typecheck.sh`）**: `Edit`/`Write`/`MultiEdit`で`.ts`/`.tsx`ファイルを編集するたびに、`chaos-ai-suite/`直下で`npm run typecheck`を実行する。エラーがあれば日本語メッセージ＋`tsc`出力の末尾を表示し、`exit 2`でClaude Codeに次の大規模編集を止めさせる。`chaos-ai-suite/packages`配下以外（ドキュメント・設定ファイル等）の編集ではスキップする
- **Stop（`stop-quality-check.sh`）**: 返答の終わりごとに、`chaos-ai-suite/packages`配下に未コミットの変更がある場合のみ、バックエンドのテスト（`npm run test --workspace packages/backend`）を実行する。Lintスクリプトが将来追加された場合は自動的に検出して実行する（現時点ではLintスクリプトがどのworkspaceにも存在しないため、実質的な処理はない）。テスト失敗時は`exit 2`で報告し、次の返答で対応を促す（無限ループ防止のため、`stop_hook_active`が立っている場合は止めない）

### 設計上の理由（頻度を分けた理由）

- 型チェックは編集のたびに実行しても数秒（キャッシュ利用時 約3.5秒）で軽いため、都度実行する
- バックエンドテストは型チェックより重く、複数ファイルを続けて編集する1回の作業の中で何度も走らせると無駄が大きいため、返答の区切り（Stop）で1回だけ実行する
- どちらも「利用可能なコマンドのみ実行する」（`package.json`を都度確認）方針を徹底しており、存在しないコマンドを推測で追加実行しない

## Subagents（`.claude/agents/`）

- **code-reviewer**: バグ・重複処理・保守性低下・既存機能を壊す可能性を確認（検出のみ、`Edit`/`Write`権限なし）
- **security-reviewer**: APIキー露出・入力値検証不足・認証漏れ・危険なコマンド・個人情報保存を確認（検出のみ）
- **mobile-ui-reviewer**: 横スクロール・文字/ボタンの見切れ・タップ領域・モーダル崩れを確認（検出のみ）

いずれも`Agent`ツールで手動的に呼び出す（Hooksのように自動発火はしない。理由: コードレビューはAI呼び出しコストが発生するため、「1操作1回呼び出し」の原則に沿い、意図的なタイミングで呼ぶ設計にした）。使うタイミングは各Skillの手順に組み込まれている。

## Skills（`.claude/skills/`）

再利用可能な作業手順を5つ用意した。ユーザーが`/<skill-name>`で呼び出すか、該当する状況でClaude Codeが自律的に選ぶ。

- **safe-feature-add**: 安全な機能追加の手順（着手前調査→実装方針→実装中の注意→実装後確認）
- **bug-investigation**: 原因調査を優先するバグ修正の手順
- **release-check**: 公開・デプロイ前の最終確認手順
- **mobile-check**: スマホUI確認の手順
- **cost-check**: API利用料が増える変更の検出手順

## 外部送信・秘密情報について

- Hookスクリプトはすべてローカルでコマンドを実行するのみで、外部サービスへの自動送信は一切行わない
- Hookの出力・サブエージェントの報告に秘密情報（APIキー等の値そのもの）を含めない設計にしている（`security-reviewer`の指示にも明記）

## 元に戻す方法

- 一時的に無効化したい場合: `.claude/settings.json`の`hooks`セクションを削除するか、ファイル自体をリネームする
- 完全に取り除きたい場合: `.claude/settings.json`・`.claude/hooks/`・`.claude/agents/`・`.claude/skills/`・このファイルをGitで削除してコミットする（アプリ本体`chaos-ai-suite/`には一切依存していないため、削除しても本体機能に影響しない）

## 既知の制約

- Lintツール（ESLint等）は現時点でこのプロジェクトに導入されていないため、自動化されていない（導入されればHookが自動検出して実行する設計にしてある）
- フロントエンドの自動テスト（vitest等）が存在しないため、Stopフックのテスト実行はバックエンドのみ。フロントエンドの動作確認は引き続き手動（ビルド成功＋実機/ブラウザ確認）が必要
- スマホでの実際の見た目崩れは`mobile-ui-reviewer`サブエージェントによるコードレビューでは検出しきれない場合がある。最終確認は引き続き実機での目視確認が必要
