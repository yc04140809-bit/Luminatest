# 01 開発ルール(基本原則)

- **最終更新**: 2026-07-16

## 基本原則

1. **理念が最上位**: すべての作業はブランド理念「安心・癒し・希望・心を整理する時間」([00章](../master-library/00_brand-philosophy.md))に従う。技術的に正しくても理念に反する変更はしない
2. **Single Source of Truth**: 設定の正は `master-library/`。同じ情報を複数の場所に書かず、リンクで参照する
3. **土台優先**: 実装(コード)より構造とドキュメントの整合性を優先する。5年後・10年後に読める状態を保つ
4. **スマホで運営可能**: すべてのデータはプレーンな Markdown / JSON で管理し、スマホのGitHubアプリ・ブラウザだけでも読み書きできる形式を守る(特殊なツールが必須になる構成にしない)
5. **小さく確定、大きく拡張**: 未確定事項は「(未設定)」と明示して枠だけ作る。勝手に埋めない

## AIエージェントの行動規範

このプロジェクトでは複数のAIが作業することを前提とします。

1. **作業前に読む**: `README.md` → `development-rules/` 全部 → 作業対象ディレクトリの `README.md`
2. **勝手に設定を創作しない**: 「(未設定)」「(仮)」の項目は、オーナーの指示があるまで確定させない。提案は提案として明示する
3. **テンプレートに従う**: 項目の追加・削除・並べ替えをしたい場合は、個別ファイルではなくテンプレート自体の改訂として提案する
4. **ID・正式表記を厳守**: Card ID / CHR-xxx / 用語集の表記を変更・省略しない
5. **破壊的変更は確認を取る**: ファイル削除・ID変更・用語廃止・キャラ設定の大変更は、実行前にオーナーの承認を得る
6. **変更を記録する**: [05_更新ルール](./05_update-rules.md) に従い履歴を残す

## 文章を書くとき

- カード・キャラのセリフ・SNS文面など、利用者の目に触れる文章はすべて [08_ライティングガイドライン](../master-library/08_writing-guidelines.md) に従う
- ドキュメント(この種の管理文書)は簡潔・明確を優先してよい

## Git 運用

- コミットメッセージは日本語可。「何を・なぜ」を1行目に書く
- 1コミット = 1つのまとまった変更(カード1枚の追加、章1つの改訂など)
- main への直接コミットは避け、作業ブランチ経由を基本とする
