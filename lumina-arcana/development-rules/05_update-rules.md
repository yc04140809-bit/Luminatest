# 05 更新ルール

- **最終更新**: 2026-07-16

設定・データを変更するときの手順と記録のルールです。
**「未来の自分と、次のAIが、変更の理由を辿れること」**が目的です。

## 更新の基本手順

1. **影響範囲を確認**: 変更対象を参照しているファイルを探す(用語ならカード・SNS文面まで)
2. **Master Library を先に更新**: 設定の正を先に直し、派生物(カード・プロンプト等)を追従させる
3. **履歴を書く**: 変更したファイル自身の「更新履歴」表に1行追加
4. **重要な変更は changelog へ**: [master-library/14_changelog.md](../master-library/14_changelog.md) にも記録
5. **コミット**: 1つのまとまった変更を1コミットにする

## 変更の重さと必要な手続き

| レベル | 例 | 手続き |
|---|---|---|
| 軽微 | 誤字修正・表現の微調整 | ファイル内履歴のみでよい |
| 通常 | カード内容の改訂・設定の追記 | ファイル内履歴+changelog |
| 重大 ⚠️ | 用語の廃止・キャラ設定の大変更・テンプレート項目の変更 | ユーザー承認 → changelog(⚠️付き)→ 影響ファイルすべて更新 |
| 禁止 | Card ID / CHR-ID の変更・再利用 | 行わない(永久不変) |

## バージョンの上げ方

- ドラフト中の改訂: `v0.1` → `v0.2`
- 正式版化: `v1.0`
- 正式版後の内容改訂: `v1.0` → `v1.1`(小)/ `v2.0`(意味が変わる改訂)

## released 済みデータの改訂

公開済み(released)のカード・設定を変える場合:

1. ステータスを `revising` にする
2. 改訂後、バージョンを上げて `released` に戻す
3. アプリ・SNS等の配信面への反映が必要かを changelog に明記する

## 定期メンテナンス

- **四半期ごと**: [13_ロードマップ](../master-library/13_roadmap.md) の見直し
- **カード10枚完成ごと**: 用語集・カードマスター台帳と実ファイルの整合チェック
- **フェーズ移行時**: development-rules 全体の見直し
