# 07 画像生成ガイドライン

- **ステータス**: ドラフト(アートスタイルの基準画像が決まり次第 v1.0 へ)
- **最終更新**: 2026-07-16

カード・キャラクター・SNS用画像を、どのAIツールで生成しても
「Lumina Arcana の絵」として揃えるためのガイドラインです。
プロンプト本文は [Prompt Library](../prompt-library/) で履歴管理します。

## アートディレクションの核

| 項目 | 方針 |
|---|---|
| 全体トーン | やわらかい光、静けさ、包まれる安心感 |
| 色 | 淡く透明感のある配色。彩度の高すぎる原色・攻撃的な赤黒は避ける |
| 光 | 光は常に「やさしく差し込む」。逆光の恐怖演出はしない |
| 描かないもの | 流血・恐怖・裁き・絶望を直接描く構図 |
| 判断基準 | 「疲れた夜に見ても心がざわつかないか」 |

## 共通スタイルプロンプト(ベース)

すべてのカード画像プロンプトの土台。変更する場合は必ず本章を先に更新する。

```
(スタイル定義・未確定 — 基準画像決定後にここへ確定版を記載)
例: soft ethereal illustration, gentle glowing light, pastel color palette,
dreamy atmosphere, warm and safe feeling, high detail, no text
```

## カード画像の仕様

| 項目 | 仕様 |
|---|---|
| アスペクト比 | 【未設定】紙カード印刷を見据え実寸比を決定する(候補: 2:3) |
| 解像度 | 印刷用マスター: 高解像度で保存。SNS/アプリ用は派生書き出し |
| 文字 | 画像内にタイトル文字を焼き込まない(多言語展開・アプリ表示のため) |
| 枠 | 共通フレームは後工程で合成(フレームデザインは未確定) |

## 生成〜採用のワークフロー

1. Prompt Library にプロンプトを新バージョンとして記録(`v001` から連番)
2. 生成した画像のうち候補を選び、生成ツール名・日付・使用プロンプト版を記録
3. 採用画像はカードファイルの「画像管理」表に登録
4. 不採用でも有望な系統はプロンプト履歴にメモを残す(再現性の確保)

## 画像ファイルの命名

```
{card-id}_v{版}.png      例: LA-M-00_v001.png
{card-id}_print_v{版}.png(印刷用マスター)
```

保存場所は `assets/`(将来作成。容量が増えたら外部ストレージ+リンク管理に移行)。

## 権利・安全

- 生成ツールの商用利用規約を確認してから採用する(ツールごとの確認結果を本章に追記)
- 実在人物・既存キャラクターに似せるプロンプトは使用しない
