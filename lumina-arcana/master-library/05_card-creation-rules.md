# 05 カード制作ルール

- **ステータス**: 正式版 v2.1(構成ルール+共通フォーマット)/ 属性・スート名は未確定
- **最終更新**: 2026-07-18

全78枚のカードを、品質と世界観を揃えて制作するためのルールです。

## デッキ構成

| 区分 | 枚数 | Card ID 範囲 | 内容 |
|---|---|---|---|
| メジャー(大アルカナ相当) | 22枚 | `LA-M-00` 〜 `LA-M-21`(0始まり) | 世界の大きなテーマ・キャラクター |
| マイナー(小アルカナ相当) | 56枚 | `LA-S1-01` 〜 `LA-S4-14` | 4スート × 14枚。日常の心の風景 |

⚠️ **番号体系(2026-07-18 確定)**: メジャーアルカナは実際のカード画像に「0」〜「21」の番号が焼き込まれているため、**0始まり・22枚**を正式採用する(伝統的な大アルカナと同じ体系)。以前「LA-M-00 をテンプレート専用予約とし LA-M-01〜LA-M-22 を正式範囲とする」としていた決定は撤回する。

- スート名(S1〜S4)は【未設定】。世界観に合わせた4属性を [01_世界観](./01_worldview.md) と整合させて決定する
- Card ID は一度発番したら**永久に変更しない**(画像・プロンプト・アプリのキーになるため)

## Card ID 規則

```
LA-M-00   … メジャー 0番(正式カードは 00 から。伝統的大アルカナと同じ0始まり)
LA-M-21   … メジャー 21番(最終番)
LA-S1-01  … スート1の1番
```

- 番号は2桁ゼロ埋め
- ファイル名は `{card-id}_{slug}.md`(例: `LA-M-00_cradle-of-life.md`)
- slug は英小文字とハイフンのみ(→ [development-rules/02_naming-conventions.md](../development-rules/02_naming-conventions.md))
- テンプレートの記入見本は実カードと区別するため `LA-TEMPLATE-00` という別識別子を使う(→ [card-database/_templates/](../card-database/_templates/))。正式カードの Card ID パターン(`LA-M-\d{2}` 等)とは一致しない

## Lumina Arcana 共通フォーマット Version 1.0(カード本文・正式仕様)

- **フォーマット版**: **Version 1.0**(正式採用 2026-07-18・ユーザー決定)
- **適用範囲**: 既存22枚(メジャーアルカナ)、および**今後追加されるすべてのカード**(マイナーアルカナ含む)、**アプリUI・JSON・データベースのすべて**を、この5項目に統一する

カード本文(文章コンテンツ)は以下5項目の共通フォーマットに統一する。

| 項目 | 英名 | 内容 | 目安 | アプリでの表示 |
|---|---|---|---|---|
| メッセージ | **Card Message** | 今日のあなたへ、短く手渡す言葉 | 1文・50字前後 | 画面①(カードとの出会い) |
| リーディング | **Reading** | カードが一人称で語りかける、意味・象徴の解説 | 詩的な複数行(目安なし) | 画面②(カードを読み解く) |
| 問いかけ | **Reflection** | 静かに自分と向き合うための問い(Yes/Noで終わらない開いた問い) | 1つ | 画面②→ジャーナルへの橋渡し |
| 今日の光 | **Today's Light** | 一人称の短い宣言文 | 1文 | 画面②(カードを読み解く) |
| キーワード | **Keywords** | カードを象徴する語 | 目安3〜5語(特別な意味を持つカードは増やしてよい。フロントマターが正) | 画面②(カードを読み解く) |

⚠️ **旧項目名「Meaning」「Affirmation」は正式に廃止**。「Reading」「Today's Light」が正式名称(ユーザー決定 2026-07-18)。card-schema.json・card-template.md・既存カードファイル・UI・JSON・データベースすべてに反映済み。

- 執筆は [08_ライティングガイドライン](./08_writing-guidelines.md) の「ブランドの声」「公式挨拶フレーズ」「NGワード」に必ず従う
- **文章コンテンツの執筆は Lumina(ブランド統括AI)/ケイオス師匠の領分**。Claude Code はフォーマット・データ構造・テンプレートの設計と、【未設定】項目の管理を担当する(→ [development-rules/06_AI Role Definition](../development-rules/06_ai-role-definition.md))
- 旧構成(ストーリー / 基本メッセージ / 恋愛・仕事・金運・人間関係 / 今日の行動 / 注意点)は 2026-07-18 に本フォーマットへ統合・置き換えられた(⚠️ 破壊的変更。理由: UX Master Specification の画面設計およびユーザー指示に合わせ、複雑な運勢別メッセージをやめ、Card Message/Reading/Reflection/Today's Light の4文章+Keywordsというシンプルな構造に統一するため)
- **Reading は「カード自身が語りかける形式」を正式仕様とする**(ユーザー制定 2026-07-18。→ [08_ライティングガイドライン](./08_writing-guidelines.md) のブランドの声を参照)

### Design Philosophy(カードデータには含めない)

各カードの特別な意味合いを説明する注釈(例: No.0「生命の揺り籠」がなぜ唯一無二のカードか)は、**カードデータ(card-database)には含めず、本章(開発ドキュメント側)でのみ管理する**(ユーザー決定 2026-07-18)。カードファイル側には本文5項目のみを置き、Design Philosophy は下表に集約する。

| Card ID | Design Philosophy |
|---|---|
| LA-M-00 | このカードは Lumina Arcana の原点である。未来を予言するカードではない。人生を変えるカードでもない。**帰る場所を思い出すカードである。** ユーザーはこのカードを通して、「ここはいつでも帰ってこられる場所」であることを最初に知る。**No.0 は、すべてのカードが生まれる世界そのものを象徴する、唯一無二のカードとする。** 文章は、カード自身が語りかける形式を正式仕様とする(この一文は全カード共通の Reading 執筆規則として [08_ライティングガイドライン](./08_writing-guidelines.md) にも反映済み)。 |

## 1枚のカードが持つ項目

Card Database のテンプレート([card-template.md](../card-database/_templates/card-template.md))に完全準拠する。
項目の追加・削除はテンプレートを先に改訂し、本章と [14_更新履歴](./14_changelog.md) に記録する。

必須項目: Card ID / 日本語タイトル / 英語タイトル / 属性 / キーワード(Keywords)/ 象徴 / **Card Message / Reading / Reflection / Today's Light**(共通フォーマット)/ ラッキーカラー / ラッキーナンバー / 画像生成プロンプト / 画像管理 / 更新履歴

## 制作フロー

1. **発番**: [06_カードマスター](./06_card-master.md) の一覧に行を追加し、Card ID を確定
2. **下書き**: テンプレート([card-template.md](../card-database/_templates/card-template.md))を複製してカードファイルを作成(status: draft)。書き方は記入見本 [card-template-filled-example.md](../card-database/_templates/card-template-filled-example.md) を参照する(内容自体は仮設定であり流用しない)
3. **文章**(Lumina/ケイオス師匠が担当): [08_ライティングガイドライン](./08_writing-guidelines.md) に従い Card Message / Reading / Reflection / Today's Light / Keywords の5項目を執筆
4. **画像**: [07_画像生成ガイドライン](./07_image-generation-guidelines.md) に従いプロンプトを Prompt Library に登録 → 生成 → 採用画像を記録
5. **レビュー**: ブランド理念・世界観・用語集との整合を確認
6. **確定**: status を released に変更し、カードマスターの一覧を更新

## 品質基準(チェックリスト)

- [ ] ブランド理念を損なう表現がない(不安を煽らない・断定しない)
- [ ] 用語が [03_用語集](./03_glossary.md) の正式表記と一致している
- [ ] Reflection が Yes/No で終わらない開いた問いになっている
- [ ] Today's Light が一人称の短い宣言文になっている
- [ ] Reading がカード自身の語りかけとして書かれている
- [ ] 画像とテキストの雰囲気が一致している
