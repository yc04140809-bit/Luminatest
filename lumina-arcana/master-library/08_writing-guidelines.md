# 08 ライティングガイドライン

- **ステータス**: 正式版 v1.3
- **最終更新**: 2026-07-18

カード文言・SNS・アプリ内テキストなど、Lumina Arcana が発するすべての言葉のルールです。
**誰が(どのAIが)書いても同じ声になる**ことを目的とします。

## ブランドの声(Voice)

- 静かで、あたたかく、そっと隣に座るような声
- 上から教えるのではなく、隣で一緒に眺める
- 押し付けない。急かさない。決めつけない
- すべての文章は「小さな希望の灯りを添える」ために書く。読み終えたとき「今日はもう少し頑張ってみよう。」「また帰ってこよう。」と自然に思えること(感情設計の核【正式設定】2026-07-18)
- **Reading は「カード自身が語りかける形式」を正式仕様とする**(ユーザー制定 2026-07-18)。カードが一人称(「私は〜」)で読み手に語りかける文体で書く

## 公式挨拶フレーズ【正式設定】(ユーザー決定 2026-07-18)

| 場面 | フレーズ | 備考 |
|---|---|---|
| 体験の開始(アプリ起動 等) | **おかえり。** | オープニングでは「おかえりなさい」(UX仕様原文)。トーンは統一 |
| 体験の終了(アプリを閉じる 等) | **いってらっしゃい。** | 小さな希望を持って日常へ送り出す言葉 |

ブランド全体(アプリ・カード・SNS・書籍)でこの思想を反映する。

## 文体ルール

| 項目 | ルール | 例 |
|---|---|---|
| 語りかけ | やさしい「です・ます」調 | 「〜かもしれません」「〜してみませんか」 |
| 断定 | 未来を断定しない | ×「必ず成功します」→ ○「うまくいく流れが近づいています」 |
| 否定 | 利用者を否定しない | ×「あなたの考えは間違いです」 |
| 命令 | 命令形を避け、提案形にする | ×「〜しなさい」→ ○「〜してみるのもいいかもしれません」 |
| 脅し | 恐怖で動かさない | ×「このままだと不幸になります」 |
| 絵文字 | SNSでは控えめに可。カード本文では使わない | |

## 項目別の書き方(カード)— Lumina Arcana 共通フォーマット(2026-07-18確定・改訂)

正式な項目定義・アプリでの表示先は [05_カード制作ルール](./05_card-creation-rules.md) の「共通フォーマット」を参照。

| 項目 | 目安 | 書き方 |
|---|---|---|
| Card Message(メッセージ) | 1文・50字前後 | カードが今日そっと手渡す言葉。画面①に表示されるため、それ単体で完結する短さにする |
| Reading(リーディング) | 詩的な複数行 | カードが一人称(「私は〜」)で語りかける形式で、象徴・情景・意味を描く。世界観(時のゆりかご)と接続し、読み手を否定せず静かに寄り添う |
| Reflection(問いかけ) | 1つ | 静かに自分と向き合うための問い。Yes/Noで終わらない開いた問いにする(例: ×「頑張っていますか?」→ ○「今日、一番大切にしたいことは何ですか?」)。ジャーナルへの導線として機能する |
| Today's Light(今日の光) | 1文 | 一人称の短い宣言文。「私は〜」で始め、断定はしても強制はしない(例: 「私は、小さな希望を信じ、ゆっくり育てます。」) |
| Keywords(キーワード) | 目安3〜5語 | カードを象徴する単語。フロントマターで管理(本文には重複記載しない)。特別な意味を持つカードは増やしてよい |

⚠️ 2026-07-18 改訂: 「Meaning」→「Reading」、「Affirmation」→「Today's Light」に名称統一(ユーザー決定)。Design Philosophy(カード固有の特別な意味合いの注釈)はカードデータには含めず、開発ドキュメント側でのみ管理する。

## NGワード・要注意表現

- 「絶対」「必ず」「二度と」「手遅れ」「最悪」「呪い」「祟り」
- 「人生が変わる」等の大きな約束(ブランドは人生を変えることを約束しない。添えるのは「小さな希望の灯り」)
- 病気・死・法律・投資について断定的な助言をしない(専門家の領域に踏み込まない)
- 他の占い・宗教・思想への言及や比較

## 表記統一

- 固有名詞は [03_用語集](./03_glossary.md) の正式表記に従う
- 数字は半角、感嘆符・疑問符は全角(!?)を基本とする
- 三点リーダーは「……」(2つ重ね)
