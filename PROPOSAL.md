# 新規自動化ツール提案書：「ケアクエスト」

Claude Code (Fable-5) を活用し、24時間以内にMVP実装・市場投入が可能な自動化ツールのアイディア提案。
本提案は、既存プロダクト「研修資料作成AI ケアちゃん」（`care-training.html`）を土台に、
先行者利益・収益化・新規性・中毒性を兼ね備えた形へ拡張するものです。

---

## 1. アイディア名と具体的コンセプト

### アイディア名：「ケアクエスト（CareQuest）」
サブタイトル：*ケアちゃんと毎日1問、15秒で介護・看護の現場力を鍛える*

### コンセプト
既存の「ケアちゃん」は、施設の研修担当者がテーマを選ぶだけで研修資料（図解・アンケート・穴埋め問題・ロープレ台本）を
自動生成する**制作ツール**である。しかし制作されたPDFは配布して終わり、現場スタッフの「継続学習」には繋がっていない。

「ケアクエスト」は、ケアちゃんが生成した穴埋め問題・ロープレ設問を自動的に**1日1問・15秒で終わるデイリークイズ**に
変換し、スマホのホーム画面（PWA）に届ける学習習慣化アプリ。Duolingo型の「ストリーク（連続正解日数）」「XP・レベル
アップ」「施設内ランキング」をケアちゃんのキャラクター演出で包み、法定研修（感染対策・虐待防止・BCP等）や
新人教育の定着率を上げる。

- **制作側（管理者）**：ケアちゃんでテーマ選択→自動で研修資料とデイリークイズが同時生成される
- **学習側（現場スタッフ）**：通知が来たら1問答えるだけ。正解でケアちゃんが喜ぶアニメーション＋連続日数が伸びる
- **施設全体**：誰が学習を続けているか、どの設問の正答率が低い（＝理解が浅い分野）かが管理者ダッシュボードで可視化

---

## 2. ターゲット市場および先行者利益の分析

### ターゲット市場
- 一次ターゲット：介護施設・訪問看護ステーション等の**人材育成担当者／施設長**（B2B導入決裁者）
- 二次ターゲット：現場スタッフ本人（パート・非正規比率が高く、離職率が高い層）

### 市場の痛み
- 介護・看護業界は深刻な人手不足と高離職率（年間離職率15%前後）を抱え、**新人の早期離脱**が経営課題
- 法定研修（虐待防止・感染対策・BCP等）は年1回の座学で「やった気」になるだけで定着しない
- 汎用LMS（ラーニングマネジメントシステム）は業界特化しておらず、UIが硬く現場スタッフに使われない

### 先行者利益
- 介護・看護特化かつ「資料生成」と「習慣化学習（ゲーミフィケーション）」を**一体化**したツールは競合が実質不在
- 既に「ケアちゃん」というキャラクターIP・トンマナ・画像資産（`img/care/avatar.png`）を保有しており、
  ブランド構築コストゼロで即座に拡張展開できる
- 介護報酬改定のたびに法定研修・記録要件が増えており、「研修コンプライアンス対応」の緊急ニーズに先んじて
  対応することで、営業初期段階から「唯一の選択肢」としてポジションを取れる

## 3. 収益化（マネタイズ）の具体的な手法

| 手法 | 内容 | 価格目安 |
|---|---|---|
| **フリーミアム（個人向け）** | 週3問まで無料。無制限利用・バッジコレクション・ケアちゃん着せ替えはサブスク | 月額480円 |
| **施設向けB2B SaaS** | 管理者ダッシュボード、研修資料生成無制限、スタッフ学習状況の可視化、CSVエクスポート | スタッフ数に応じ月額5,000〜50,000円 |
| **認定証・修了証発行** | 法定研修の受講証跡としてPDF証明書を発行（監査・実地指導対策） | 1件330円、または施設契約に内包 |
| **業界特化コンテンツパック** | 感染症流行期の特別問題集、介護報酬改定対応クイズ等を都度販売 | 1パック980円〜 |

広告・個人データ販売は行わない（要配慮個人情報を扱う業界特性上、信頼性を毀損するリスクを回避）。

## 4. Claude Codeを用いた自動化実装のステップと重要ポイント

### 実装ステップ（24時間MVP）

1. **既存資産の転用（1〜2時間）**
   Claude Codeに `care-training.html` の穴埋め問題生成ロジック・キャラクターUI・カラートークン
   （teal `#1f7a72` / accent `#f28c38`）を読み込ませ、そのまま「デイリークイズ」画面のベースにする。
2. **クイズデータ構造の設計（1時間）**
   1問＝「question / choices / answer / explanation / category」のJSONを生成するプロンプトテンプレートを
   Claude Codeに実装させ、ケアちゃんの資料生成フローの最後に自動で1問抽出する処理を追加。
3. **ゲーミフィケーションUI（4〜5時間）**
   - ストリーク・XP・レベルは**バックエンド無し**でまず `localStorage` に実装（サーバー不要でMVP最速化）
   - 正解時のキャラクターアニメーション（既存 `charavatar` を流用し弾む・光るCSSアニメーション追加）
4. **PWA化・通知対応（2〜3時間）**
   `manifest.json` と Service Worker を追加し、ホーム画面追加とプッシュ通知（毎日決まった時刻にリマインド）を実装。
5. **施設内ランキング（3〜4時間）**
   MVPはFirebase Realtime Database等の無料枠を使い、Claude Codeにセキュリティルール込みで雛形生成させる。
   施設IDごとにスコアを集計し、上位スタッフを表示。
6. **管理者ダッシュボード（3〜4時間）**
   誰が何日連続学習しているか、正答率が低い分野は何かを一覧表示する画面を追加。
7. **デプロイ（1時間）**
   GitHub Pages / Vercel 等、静的ホスティングで即時公開。ドメイン取得は後回しにし、まず動くものを世に出す。

### 重要ポイント
- **1ファイル完結の哲学を守る**：既存リポジトリのHTML単一ファイル構成を踏襲し、ビルド工程を持たないことで
  Claude Codeによる高速な反復開発（生成→ブラウザ確認→修正）を可能にする。
- **バックエンドは最小限に絞る**：認証・DBはFirebase無料枠に限定し、24時間という制約の中では
  「習慣化を演出するフロントエンドの体験」に開発リソースを最優先で投下する。
- **既存キャラクターIPを流用し新規デザインコストをゼロにする**：ケアちゃんの画像・トンマナをそのまま使うことで、
  デザイン検討の時間を機能実装に回せる。
- **法定研修という"待ったなし"の需要にフックする**：単なる娯楽的ゲーミフィケーションではなく、
  監査対応・実地指導対策という施設側の切実なニーズと結びつけることで、「使われないゲーム」で終わらせない。
